Installation

Get started with InferenceIQ in minutes. Install the Python SDK from PyPI.

Via pip

The easiest way to install InferenceIQ is using pip:

pip install inferenceiq

From source

To install from the GitHub repository:

git clone https://github.com/awh233/inferenceiq.git
cd inferenceiq
pip install -e .

Verify Installation

Verify your installation by checking the version:

python -c "import inferenceiq; print(inferenceiq.__version__)"

API Key Setup

To use InferenceIQ, you'll need an API key. Get one from the InferenceIQ Dashboard.

export INFERENCEIQ_API_KEY="your_api_key_here"

Or set it in your code:

import os
os.environ["INFERENCEIQ_API_KEY"] = "your_api_key_here"
Tip: Keep your API key secure and never commit it to version control. Use environment variables or .env files.

Next Steps

After installation, check out the First API Call guide to make your first request.

Your First API Call

Make your first API call to InferenceIQ in under a minute.

Basic Usage

Initialize the client and make a simple request:

from inferenceiq import InferenceIQ

client = InferenceIQ(api_key="your_api_key")

# Make an inference request
response = client.infer(
    model="gpt-4-turbo",
    messages=[
        {"role": "user", "content": "What is 2+2?"}
    ],
    routing_strategy="cost_optimized"
)

print(response.result)

Understanding the Response

The response includes:

Field Description
result The generated text or response from the model
model_used The actual model that processed the request
cost The estimated cost of this inference in USD
latency_ms Time taken to process the request in milliseconds
cached Whether the result was retrieved from the semantic cache

Routing Strategies

InferenceIQ supports multiple routing strategies to optimize for your needs:

# Cost-optimized: Use cheaper models when possible
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Hello"}],
    routing_strategy="cost_optimized"
)

# Quality-first: Prefer higher quality models
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Hello"}],
    routing_strategy="quality_first"
)

# Balanced: Balance between cost and quality
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Hello"}],
    routing_strategy="balanced"
)

# Latency-first: Minimize response time
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Hello"}],
    routing_strategy="latency_first"
)

Error Handling

Handle errors gracefully:

from inferenceiq import InferenceIQ
from inferenceiq.exceptions import InferenceIQError

client = InferenceIQ(api_key="your_api_key")

try:
    response = client.infer(
        model="gpt-4-turbo",
        messages=[{"role": "user", "content": "Hello"}]
    )
except InferenceIQError as e:
    print(f"Error: {e}")
    print(f"Status code: {e.status_code}")

Python SDK Reference

InferenceIQ Client

The main client class for synchronous operations.

Initialization

from inferenceiq import InferenceIQ

client = InferenceIQ(

infer() Method

Make an inference request.

response = client.infer(
    model: str,                          # Model name (required)
    messages: List[Dict[str, str]],      # Chat messages (required)
    routing_strategy: str = "balanced",  # Routing strategy (optional)
    temperature: float = 0.7,            # Sampling temperature (optional)
    max_tokens: int = 2048,              # Max output tokens (optional)
    top_p: float = 1.0,                  # Nucleus sampling param (optional)
    frequency_penalty: float = 0.0,      # Frequency penalty (optional)
    presence_penalty: float = 0.0,       # Presence penalty (optional)
    stop: List[str] = None               # Stop sequences (optional)
) -> InferenceResponse

Async Client

Use the InferenceIQAsync client for non-blocking operations in async applications.

Initialization

from inferenceiq import InferenceIQAsync
import asyncio

async def main():
    client = InferenceIQAsync(api_key="your_api_key")

    response = await client.infer(
        model="gpt-4-turbo",
        messages=[{"role": "user", "content": "Hello"}]
    )

    print(response.result)

asyncio.run(main())

Concurrent Requests

Run multiple inferences concurrently:

import asyncio
from inferenceiq import InferenceIQAsync

async def main():
    client = InferenceIQAsync(api_key="your_api_key")

    # Create multiple tasks
    tasks = [
        client.infer(
            model="gpt-4-turbo",
            messages=[{"role": "user", "content": f"Question {i}"}]
        )
        for i in range(5)
    ]

    # Run them concurrently
    responses = await asyncio.gather(*tasks)

    for response in responses:
        print(response.result)

asyncio.run(main())

Context Manager

Use the async client with a context manager for proper cleanup:

from inferenceiq import InferenceIQAsync
import asyncio

async def main():
    async with InferenceIQAsync(api_key="your_api_key") as client:
        response = await client.infer(
            model="gpt-4-turbo",
            messages=[{"role": "user", "content": "Hello"}]
        )
        print(response.result)

asyncio.run(main())

Streaming Responses

Stream responses asynchronously:

async def main():
    client = InferenceIQAsync(api_key="your_api_key")

    async for chunk in client.infer_stream(
        model="gpt-4-turbo",
        messages=[{"role": "user", "content": "Write a story"}]
    ):
        print(chunk, end="", flush=True)

asyncio.run(main())

Error Handling

Handle async errors:

from inferenceiq import InferenceIQAsync

Types & Models

Complete reference for InferenceIQ data types and models.

InferenceResponse

The response returned from inference operations.

from inferenceiq.types import InferenceResponse

class InferenceResponse:
    """Response from an inference request"""
    result: str              # Generated text output
    model_used: str          # Actual model used (may differ from requested)
    cost: float              # Cost in USD
    latency_ms: int          # Response time in milliseconds
    cached: bool             # Whether retrieved from semantic cache
    provider: str            # Model provider (openai, anthropic, etc)
    input_tokens: int        # Tokens in the input
    output_tokens: int       # Tokens in the output
    total_tokens: int        # Total tokens used
    stop_reason: str         # Why generation stopped
    metadata: Dict[str, Any] # Additional metadata

Routing Strategies

InferenceIQ intelligently routes your requests to different models based on your optimization preferences.

Overview

Choose the routing strategy that best matches your use case. InferenceIQ analyzes request complexity and automatically selects the optimal model.

Cost Optimized

Minimize API costs by routing to cheaper models when quality permits. Great for high-volume inference.

Quality First

Prioritize response quality by selecting the most capable models. Best for critical tasks.

Balanced

Default strategy balancing cost and quality. Recommended for most use cases.

Latency First

Minimize response time. Perfect for real-time applications and user-facing features.

Cost Optimized

This strategy analyzes your request and routes to the cheapest model capable of handling it effectively.

response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Translate to Spanish: Hello"}],
    routing_strategy="cost_optimized"
)
# Might use: gpt-3.5-turbo (saves ~70% cost)

When to Use

  • High-volume inference workloads
  • Simple, straightforward tasks
  • Cost-sensitive applications
  • Background processing

Semantic Caching

InferenceIQ uses semantic caching to dramatically reduce costs and latency for similar requests.

How It Works

Instead of exact text matching, semantic caching uses embeddings to understand request meaning. If you ask a similar question with different wording, we can return the cached result.

# First request - goes to the API
response1 = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "What is machine learning?"}]
)
print(f"Cost: ${response1.cost}, Cached: {response1.cached}")
# Cost: $0.05, Cached: False

# Very similar request - uses cache!
response2 = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "Can you explain machine learning?"}]
)
print(f"Cost: ${response2.cost}, Cached: {response2.cached}")
# Cost: $0.0001, Cached: True (99.8% cost savings!)

Configuration Options

TTL (Time-To-Live)

Control how long cached results are valid (default: 1 hour).

# Cache results for 24 hours
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "..."}],
    cache_ttl_seconds=86400  # 24 hours
)

# Disable caching for this request
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "..."}],
    cache_enabled=False
)

Similarity Threshold

Control how similar a request must be to get a cache hit (0-1, default: 0.95).

# Stricter matching (95%+ similarity required)
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "..."}],
    cache_similarity_threshold=0.95
)

# Looser matching (80%+ similarity sufficient)
response = client.infer(
    model="gpt-4-turbo",
    messages=[{"role": "user", "content": "..."}],
    cache_similarity_threshold=0.80
)

Self-Hosting Guide

Run InferenceIQ on your own infrastructure for maximum control and privacy.

Architecture

InferenceIQ consists of three main components:

  • API Server: Handles inference requests and routing
  • Cache Service: Semantic caching backend (Redis-compatible)
  • Analytics Service: Usage tracking and billing

Local Development Setup

Prerequisites

Python 3.9+
Docker and Docker Compose (optional but recommended)
Redis 6.0+ (for caching)
PostgreSQL 12+ (for analytics)

Installation

# Clone the repository
git clone https://github.com/awh233/inferenceiq.git
cd inferenceiq

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
pip install -r requirements-dev.txt

# Create .env file
cp .env.example .env

Configuration

Edit .env with your settings:

# API Keys
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...

# Database
DATABASE_URL=postgresql://user:password@localhost:5432/inferenceiq

# Redis Cache
REDIS_URL=redis://localhost:6379

# Server
API_PORT=8000
API_HOST=0.0.0.0
DEBUG=false

# Routing
DEFAULT_ROUTING_STRATEGY=balanced
ENABLE_COST_OPTIMIZATION=true

# Cache
CACHE_TTL_SECONDS=3600
CACHE_SIMILARITY_THRESHOLD=0.95

Running Locally

# Start all services with Docker Compose
docker-compose up -d

# Or run individually:
# Terminal 1: Start Redis
redis-server

# Terminal 2: Start PostgreSQL
docker run -d -e POSTGRES_PASSWORD=password -p 5432:5432 postgres:15

Render Deployment

Create Render Account

  • Go to render.com
  • Sign up with GitHub
  • Connect your InferenceIQ repository

Deploy API Service

1. Create New > Web Service
2. Connect GitHub repository
3. Set Runtime: Python 3.11
4. Build Command: pip install -r requirements.txt
5. Start Command: gunicorn -w 4 -k uvicorn.workers.UvicornWorker inferenceiq.api:app
6. Add Environment Variables:
   - OPENAI_API_KEY
   - ANTHROPIC_API_KEY
   - DATABASE_URL (use Render PostgreSQL)
   - REDIS_URL (use Render Redis)
7. Deploy

Add Database

1. Create New > PostgreSQL
2. Copy connection string to DATABASE_URL
3. Run migrations: python -m alembic upgrade head

Add Redis Cache

1. Create New > Redis
2. Copy connection string to REDIS_URL
3. Restart API service

Code Examples

Practical examples for common use cases with InferenceIQ.

Synchronous Inference

from inferenceiq import InferenceIQ

client = InferenceIQ(api_key="your_api_key")

# Simple question
response = client.infer(
    model="gpt-4-turbo",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Explain quantum computing in simple terms."}
    ]
)

print(f"Response: {response.result}")
print(f"Cost: ${response.cost}")
print(f"Model: {response.model_used}")
print(f"Latency: {response.latency_ms}ms")

Asynchronous Inference

import asyncio
from inferenceiq import InferenceIQAsync

async def main():
    client = InferenceIQAsync(api_key="your_api_key")

    response = await client.infer(
        model="gpt-4-turbo",
        messages=[
            {"role": "user", "content": "What is machine learning?"}
        ]
    )

    print(response.result)

asyncio.run(main())

Concurrent Requests

import asyncio
from inferenceiq import InferenceIQAsync

async def process_questions(questions):
    client = InferenceIQAsync(api_key="your_api_key")

    tasks = [
        client.infer(
            model="gpt-4-turbo",
            messages=[{"role": "user", "content": q}]
        )
        for q in questions
    ]

    results = await asyncio.gather(*tasks)
    return results

questions = [
    "What is AI?",
    "Explain machine learning",
    "What are neural networks?"
]

responses = asyncio.run(process_questions(questions))
for i, response in enumerate(responses):
    print(f"Q: {questions[i]}")
    print(f"A: {response.result}\n")

Using Routing Strategies

from inferenceiq import InferenceIQ

client = InferenceIQ(api_key="your_api_key")

message = {"role": "user", "content": "What is 2+2?"}

# Cost optimized
cheap = client.infer(
    model="gpt-4-turbo",
    messages=[message],
    routing_strategy="cost_optimized"
)
print(f"Cost optimized: ${cheap.cost}, model: {cheap.model_used}")

# Quality first
quality = client.infer(
    model="gpt-4-turbo",
    messages=[message],
    routing_strategy="quality_first"
)
print(f"Quality first: ${quality.cost}, model: {quality.model_used}")

# Balanced (default)
balanced = client.infer(
    model="gpt-4-turbo",
    messages=[message],
    routing_strategy="balanced"
)
print(f"Balanced: ${balanced.cost}, model: {balanced.model_used}")

# Latency first
fast = client.infer(
    model="gpt-4-turbo",
    messages=[message],
    routing_strategy="latency_first"
)
print(f"Latency first: {fast.latency_ms}ms, model: {fast.model_used}")

Error Handling

from inferenceiq import InferenceIQ
from inferenceiq.exceptions import (
    AuthenticationError,
    RateLimitError,
    ModelNotFoundError,
    InferenceIQError
)

client = InferenceIQ(api_key="your_api_key")

try:
    response = client.infer(
        model="gpt-4-turbo",
        messages=[{"role": "user", "content": "Hello"}]
    )
except AuthenticationError:
    print("Invalid API key")
except RateLimitError:
    print("Rate limit exceeded, retry later")
except ModelNotFoundError:
    print("Model not available")
except InferenceIQError as e:
    print(f"API error: {e.message}")
    print(f"Status code: {e.status_code}")

FAQ Chatbot

from inferenceiq import InferenceIQ

class FAQBot:
    def __init__(self, api_key):
        self.client = InferenceIQ(api_key=api_key)
        self.system_prompt = """You are a helpful FAQ assistant.
Keep responses concise and clear. If you don't know the answer,
say so and suggest contacting support."""

    def answer(self, question):
        response = self.client.infer(
            model="gpt-4-turbo",
            messages=[
                {"role": "system", "content": self.system_prompt},
                {"role": "user", "content": question}
            ],
            routing_strategy="cost_optimized"  # Save money on FAQs
        )
        return {
            "answer": response.result,
            "cost": response.cost,
            "cached": response.cached
        }

bot = FAQBot(api_key="your_api_key")

# Questions (similar ones will hit the cache)
questions = [
    "How do I reset my password?",
    "How do I change my password?",
    "What's your refund policy?"
]

for q in questions:
    result = bot.answer(q)
    print(f"Q: {q}")
    print(f"A: {result['answer']}")
    print(f"Cached: {result['cached']}\n")

Using Different Models

from inferenceiq import InferenceIQ

client = InferenceIQ(api_key="your_api_key")

models_to_test = [
    "gpt-4-turbo",
    "gpt-4",
    "gpt-3.5-turbo",
    "claude-3-opus",
    "claude-3-sonnet"
]

prompt = {"role": "user", "content": "What is neural networks?"}

for model in models_to_test:
    try:
        response = client.infer(model=model, messages=[prompt])
        print(f"{model:20} | Cost: ${response.cost:.4f} | Latency: {response.latency_ms}ms")
    except Exception as e:
        print(f"{model:20} | Error: {e}")
